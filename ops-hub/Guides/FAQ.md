# ❓ Frequently Asked Questions

### Is this real? Are people betting real money on FedEx packages?
**No.** It is paper-only with made-up data. Full detail: [[Safety & What This Is NOT]].

### Do I need to be technical to use this hub?
**No.** This hub is written for non-technical ops managers. The app itself is
usually started by a technical teammate; you just need to know how to *use* and
*explain* it. See [[Run a Demo (step by step)]].

### Do I need to install anything?
To read this hub nicely, install **Obsidian** (free) — see [[Setup]].
To *run the app*, a technical teammate sets it up once. You don't have to.

### What is Obsidian and why are we using it?
Obsidian turns this folder of plain-text files into a clean, clickable handbook.
We use it because the files are **plain text** (they'll never be locked into one
vendor), it's **free**, and it works the same on **Windows, Mac, and Linux**, and
syncs over **OneDrive or Google Drive**.

### Will I break anything by clicking around in here?
No. Reading and clicking links is completely safe. If you edit a note, the worst
case is you change some text — and Obsidian keeps a version history
(`File recovery`) so it can be undone.

### The dashboards look like raw code / a gray box. Is it broken?
Not broken — they just need the **Dataview** add-on enabled once.
See [[Enable the Dashboards (Dataview)]] (2 minutes).

### How do I show this to someone?
Follow [[Run a Demo (step by step)]] and use the [[Daily Demo Checklist]].

### Something's not working during a demo. What do I do?
Go to [[Troubleshooting]]. If it looks serious, [[Incident & Escalation]].

### Where are the real engineering documents?
In the code repository's `docs/` folder. This hub links to them where useful —
see [[Where the technical docs live]].

### Can I add my own notes, meeting minutes, or questions?
**Yes, please.** That's the point. Use a ready-made template for consistent notes —
[[Templates/Demo Session Log|Demo Session Log]], [[Templates/Decision Entry|Decision Entry]],
or [[Templates/Meeting Note|Meeting Note]] — or just create a new note anywhere. Add
open questions to [[Decisions & Open Questions]].
